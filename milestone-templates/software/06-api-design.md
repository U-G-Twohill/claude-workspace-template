# Milestone: API Design
> Type: milestone-template
> Category: software
> Order: 6
> Optional: false

## Goal
Public interfaces are well-defined with clear contracts, consistent patterns, and proper error handling.

## Tasks
- [ ] Define public API surface (functions, endpoints, CLI commands)
- [ ] Document input/output contracts for each interface
- [ ] Design consistent error handling and error codes
- [ ] Write usage examples for each public API
- [ ] Review API for consistency (naming, parameter order, return types)
- [ ] Get feedback on API design before locking it down
- [ ] Run /document api to generate API documentation

## Notes
API design is hard to change later. Spend time here getting the interface right before building out the implementation. Prefer convention over configuration.
